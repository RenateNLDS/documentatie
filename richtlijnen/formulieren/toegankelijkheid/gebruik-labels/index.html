Toegankelijkheid

Een toegankelijk formulier begint met goede labels

Zorg dat een gebruiker de vraag begrijpt. Het tekstlabel dient zo duidelijk en beschrijvend mogelijk te zijn en waar nodig kan er extra hulp geboden worden. Zie het onderwerp Hulp en ondersteuning voor meer informatie hierover.

Raadpleeg de algemene content richtlijnen voor taal en taalgebruik.

Niet doen: Ingewikkeld label

Door veel informatie in te korten wordt het label onduidelijk

<label for="name">Voornaam(en)</label>
<input
  type="text"
  name="first_name"
  id="name"
/>

Doen: Simpel label met extra beschrijving

Gebruik waar nodig een lossedescription om meer uitleg te geven.

Vul de voornaam of voornamen in zoals deze op je id-kaart of paspoort staan.
<label for="first-name">Voornaam(en)</label>
<p id="first-name-description">
  Vul de voornaam of voornamen in zoals deze op je id-kaart of paspoort staan.
</p>
<input
  type="text"
  name="first_name"
  id="first-name"
  aria-describedby="first-name-description"
/>

Gebruik de juiste code

Maak je formulier meer toegankelijk door het <label> element op de juiste manier in te zetten.

Het <label> element is niet alleen bedoeld om de bedoeling van de inhoud te beschrijven, maar er zit ook ingebouwde toegankelijkheid in die je met geen enkel ander element zo makkelijk kan bereiken. Elk zichtbaar formulier element moet daarom een bijbehorend <label> element hebben. Voor mensen die het formulier gebruiken zonder de zichtbare pagina context is het invullen van elk formulierveld waar een gekoppeld label aan zit stukken makkelijker.

Ook kun je op het label klikken om automatisch het juiste formulierveld te selecteren.

Veel gemaakte fouten

Niet doen: Helemaal geen label

Zonder label is het voor niemand duidelijk
<input
  type="text"
  name="first_name"
/>

Niet doen: Placeholder in plaats van label

Met alleen een placeholder is het niet meer duidelijk waar de input voor is als de gebruiker begint met invullen. Ook lezen niet alle screen-readers de placeholder voor waardoor het net zo slecht werkt als zonder label
<input
  type="text"
  name="first_name"
  placeholder="Voornaam"
/>

Niet doen: Label zonder gekoppelde input

Door het label element te gebruiken zonder for mis je de koppeling tussen het label en de input voor mensen die de visuele context niet waarnemen.

<label>Voornaam</label>
<input
  type="text"
  name="first_name"
/>

Niet doen: Verkeede HTML element voor het label

Door geen label element te gebruiken mis je de ingebouwde logica die zorgt dat hij als label kan worden geinterpreteerd. Er bestaat zo geen enkele koppeling tussen het label en de input voor mensen die de visuele context niet waarnemen.

Voornaam

<p for="name">Voornaam</p>
<input
  type="text"
  name="first_name"
  id="name"
/>

Niet doen: Input ID niet uniek

Door het id van de input dubbel te gebruiken krijg je niet de juiste koppeling tussen het label en de input voor mensen die de visuele context niet waarnemen.

<label for="name">Voornaam</label>
<input
  type="text"
  name="first_name"
  id="name"
/>
<label for="name">Achternaam</label>
<input
  type="text"
  name="first_name"
  id="name"
/>

Juiste manier van een label

Doen: Gekoppeld label met een duidelijke naam

Door een label te gebruiken met een duidelijk naam en de input te voorzien van een id attribuut kan het label aan de input gekoppeld worden met het htmlFor attribuut

<label for="357f1ba2-0c4a-4cbb-b393-3c474d9a8de9">Voornaam</label>
<input
  type="text"
  name="first_name"
  id="357f1ba2-0c4a-4cbb-b393-3c474d9a8de9"
/>

Doen: Elke checkbox een eigen gekoppeld label

Door een label te gebruiken voor elke checkbox en deze met een uniekeid en for attribuut aan elkaar te koppelen werken de inputs voor iedereen. Een fieldset met legend helpen de checkboxes als geheel te herkennen.

Hoe wil je dat wij contact opnemen?

<fieldset>
  <legend>Hoe wil je dat wij contact opnemen?</legend>
  <label for="phone">
    <input
      id="phone"
      name="phone"
      type="checkbox"
      value="true"
    />
    Telefoon
  </label>
  <label for="email">
    <input
      id="email"
      name="email"
      type="checkbox"
      value="true"
    />
    E-mail
  </label>
  <label for="mail">
    <input
      id="mail"
      name="mail"
      type="checkbox"
      value="true"
    />
    Brief
  </label>
</fieldset>
